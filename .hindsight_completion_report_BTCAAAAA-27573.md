## Data Flow

```
Fix/bug issue created
    ↓
Manually marked "done" (or transitioned via Impact Gate worker)
    ↓
GitHub Actions cron fires (every 5 min)
    ↓
scan_fix_issues_done.py runs
    ├─ Query Paperclip API for all done issues
    ├─ Filter for fix/bug labels
    ├─ Check .impact_gate_muted_state.json for prior results
    ├─ Identify ungated issues
    └─ If retroactive=true:
        ├─ Run Impact Gate worker on each ungated issue
        ├─ Post results comments
        ├─ Save muted state entries
        └─ Update metrics
    ↓
Commit snapshot & muted state to git
    ↓
Health check runs (every 10 min)
    ├─ Verify snapshot freshness
    ├─ Calculate coverage %
    ├─ Check error/fail rates
    └─ Alert if any threshold violated
    ↓
If ungated issues detected:
    └─ scan_done_alert.py creates CTO issue
```

## Key Decisions

1. **Dual execution paths:** GitHub Actions (primary, scalable) + systemd (optional local fallback)
2. **Muted state file:** Local `.json` file for persistence; simpler than database, fast lookups
3. **In-memory deduplication:** Polling worker uses set per cycle; avoids redundant comments on done issues
4. **Done-guard pattern:** Three-layer check (client, worker, blast-radius) prevents reopen loops
5. **Retroactive gating:** Runs only on ungated issues; muted state prevents redundant work
6. **Daily snapshots:** Committed to git for audit trail and blame visibility
7. **Health monitoring:** Separate 10-min check ensures daemon health without blocking scan

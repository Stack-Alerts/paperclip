## How It Works (End-to-End)

```
Every 5 minutes:
  1. GitHub Actions triggers `impact-gate-polling-daemon.yml`
  2. Daemon checks out repo and installs dependencies
  3. Python script scans done fix/bug issues from last 10 minutes
  4. For each ungated issue:
       a. Fetches touched files from description (or git fallback)
       b. Queries Blast Radius for FR impact set and regression bugs
       c. Renders verification comment with file list and impact
       d. Posts comment to issue (idempotent, with scan-done marker)
  5. Returns JSON summary with gated/skipped/error counts
  6. GitHub Actions logs results to step summary
  7. Daily snapshot captures coverage metrics
  8. If ungated_count > 0, CTO alert created
```

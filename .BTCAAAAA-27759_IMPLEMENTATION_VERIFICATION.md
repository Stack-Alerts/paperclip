### 6. Integration Points ✅

**Impact Gate Worker Integration:**
- Calls `process_issue(issue_id, dry_run=False, force=True)`
- Posts detailed gate result comments to issues
- Transitions issues based on gate outcome:
  - PASS → stays done
  - FAIL → transitions to in_progress
  - ERROR/SKIPPED → muted for retry

**Muted State Management:**
- File: `.impact_gate_muted_state.json`
- Tracks issues that were gated in mute mode (avoiding reopen loops)
- Supports selective retry (--retry-errors, --retry-fails)
- Automatically committed to Git

**Paperclip API Integration:**
- Uses `_paginate()` for efficient issue listing
- Fetches issue comments for gate status detection
- Supports regex patterns for gate result matching
- Handles API pagination and error recovery

## Operational Flow

```
Every 5 minutes (GitHub Actions scheduled):
├─ Fetch all done issues from Paperclip API
├─ Filter to fix/bug issues only
├─ Optionally filter by days_back (default: 7 days)
├─ For each issue:
│  ├─ Check comments for Impact Gate result
│  ├─ Check muted gate state file
│  ├─ Categorize as gated or ungated
│  └─ Track metrics (PASS, FAIL, ERROR, SKIPPED, BYPASSED)
├─ Generate JSON summary with:
│  ├─ Timestamp and configuration
│  ├─ Total counts and categorization
│  ├─ Per-issue details
│  └─ Last 24-hour metrics
├─ If retroactive mode:
│  ├─ For each ungated issue:
│  │  ├─ Run full Impact Gate via process_issue()
│  │  ├─ Post detailed result comment
│  │  ├─ Transition issue if needed (FAIL → in_progress)
│  │  └─ Track muted results
│  └─ Update counts from retroactive results
├─ Upload JSON artifact (30-day retention)
├─ Write markdown step summary
├─ Create alert issue if gaps detected
├─ Write data quality snapshot (JSON)
├─ Commit changes if snapshot/muted state changed
└─ Push to repository
```

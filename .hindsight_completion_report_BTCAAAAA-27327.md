## Pipeline Architecture

- **Cron**: `*/5 * * * *` via GitHub Actions schedule trigger
- **Gate logic**: For each done fix/bug issue, verifies fix commits reference the issue identifier and touch source files
- **Retroactive mode**: `--retroactive` gates previously-ungated issues with `force=True` (muted, no reopen)
- **Health check**: `*/10 * * * *` verifies snapshot freshness (<15min), coverage (>90%), error/fail rates (<35%)
- **Alerting**: When ungated > 0, creates a `medium`-priority issue assigned to CTO
- **Retry**: `--retry-errors` and `--retry-fails` purge muted state entries for fresh gating
- **Concurrency**: Group `impact-gate-scan-done` with `cancel-in-progress: false`

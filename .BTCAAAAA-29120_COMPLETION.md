## How It Works

1. **Polling** (every 5 min):
   - Queries Paperclip for done fix/bug issues completed in last 10 min
   - Deduplicates across cycles

2. **Gating** (on each ungated issue):
   - Calls `impact_gate.worker.process_issue()` with force=True
   - Executes full Impact Gate (FR acceptance + regression tests)
   - Receives gate_status: PASS, FAIL, ERROR, or SKIPPED

3. **Blocking** (on test failure):
   - Creates blocking issues in Paperclip
   - Prevents promotion until tests pass

4. **Logging** (to systemd):
   - Structured JSON output per cycle
   - Indexed by `journalctl -u paperclip-impact-gate-scan-done.service`

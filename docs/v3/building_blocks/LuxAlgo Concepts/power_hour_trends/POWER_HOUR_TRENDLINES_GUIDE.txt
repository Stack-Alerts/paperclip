WHAT IS POWER HOUR TRENDLINES?

The Power Hour (final hour of trading session) is recognized as:
✓ Highest institutional activity (portfolio rebalancing)
✓ Market on Close (MOC) orders concentrated
✓ Best price discovery for high-value positions
✓ Most reliable trend signals

Power Hour Trendlines extracts ONLY prices from this premium window,
then applies linear regression to create:
✓ Middle trendline (all closes)
✓ Upper trendline (bullish breakouts)
✓ Lower trendline (bearish breakouts)
✓ Dynamic support/resistance channels

QUICK START (5 MINUTES)

from luxalgo_power_hour_trendlines import PowerHourTrendlines
from luxalgo_power_hour_trendlines_advanced import (
    BreakoutDetector, TrendStrengthAnalyzer
)
import pandas as pd

# Load hourly OHLCV data
df = pd.read_csv('spy_1h.csv', index_col='timestamp', parse_dates=True)

# Create power hour analyzer (15:00-16:00 NY time by default)
pht = PowerHourTrendlines(
    start_hour=15,
    start_minute=0,
    end_hour=16,
    end_minute=0,
    sessions_memory=20,  # Use last 20 power hours
)

# Analyze
df_result, analyses = pht.analyze(df)

print(f"✓ Power hours: {len(analyses)}")

if analyses:
    latest = analyses[-1]
    print(f"Trend: {latest.trend_direction.value}")
    print(f"Resistance: ${latest.resistance_level:.2f}")
    print(f"Support: ${latest.support_level:.2f}")
    print(f"Volatility: {latest.volatility_regime.value}")

# Detect breakouts
breakouts = BreakoutDetector.detect_breakouts(
    analyses, df['close']
)
print(f"✓ Breakouts: {len(breakouts)}")

# Analyze trend strength
if analyses:
    strength = TrendStrengthAnalyzer.analyze_trend_strength(
        analyses[-1]
    )
    print(f"Trend strength: {strength['overall_strength']:.0f}%")

HOW IT WORKS (6 STEPS)

Step 1: EXTRACT POWER HOURS
  Filter OHLCV data for 15:00-16:00 NY time
  Default configurable to any session timing
  
Step 2: GROUP BY SESSION
  Group power hour bars by trading date
  Typically 1 bar per session (1H timeframe)
  
Step 3: BUILD MIDDLE TRENDLINE
  Linear regression on all closes
  Determines primary trend direction
  Calculates R-squared confidence
  
Step 4: BUILD UPPER TRENDLINE
  Linear regression on closes ABOVE middle
  Captures bullish volatility envelope
  Defines resistance channel
  
Step 5: BUILD LOWER TRENDLINE
  Linear regression on closes BELOW middle
  Captures bearish volatility envelope
  Defines support channel
  
Step 6: ANALYZE METRICS
  Calculate channel width (volatility)
  Determine trend direction and strength
  Extract support/resistance levels
  Detect breakouts and reversals

TREND INTERPRETATION

UPTREND (Positive Slope)
  ✓ Resistance rising
  ✓ Support rising
  ✓ Channel tilted up
  Strategy: Buy dips to lower trendline
  Target: Upper trendline
  Stop: Below support

DOWNTREND (Negative Slope)
  ✓ Resistance falling
  ✓ Support falling
  ✓ Channel tilted down
  Strategy: Sell bounces to upper trendline
  Target: Lower trendline
  Stop: Above resistance

RANGING (Flat Slope)
  ✓ Resistance steady
  ✓ Support steady
  ✓ Parallel channel
  Strategy: Buy support, sell resistance
  Range trades with tight risk/reward
  Break above resistance or below support

═══════════════════════════════════════════════════════════════

VOLATILITY INTERPRETATION

Channel Width = Upper - Lower Trendline

NARROW CHANNEL (<0.5% of price)
  ✓ Low volatility
  ✓ Tight consolidation
  ✓ Big move coming soon
  Strategy: Prepare for breakout
  Entry: At channel extremes
  Stop: Tight, inside channel

MODERATE CHANNEL (0.5-1%)
  ✓ Normal volatility
  ✓ Healthy trending
  ✓ Good trading range
  Strategy: Trade channel bounces
  Entry: At trendlines
  Stop: Outside channel

WIDE CHANNEL (>1%)
  ✓ High volatility
  ✓ Strong moves
  ✓ Institutional activity
  Strategy: Breakout trading
  Entry: Outside channel
  Stop: Inside channel

EXTREME CHANNEL (>2%)
  ✓ Market uncertainty
  ✓ Shock events
  ✓ Gap risk
  Strategy: Reduce position size
  Wait for consolidation
  Avoid range trades

PARAMETERS & CONFIGURATION

POWER HOUR TIMING
  start_hour: Hour to begin (0-23, 24hr format NY time)
  start_minute: Minute to begin
  end_hour: Hour to end
  end_minute: Minute to end
  
  Default: 15:00-16:00 (US market close)
  
  Other options:
  - 08:30-09:30 (US market open)
  - 13:00-14:00 (afternoon session)
  - 14:00-15:00 (pre-close)

SESSIONS_MEMORY
  Default: 20 (last 20 power hour sessions)
  
  5-10 sessions: Responsive, recent data only
  20 sessions: Balanced (recommended)
  40-50 sessions: Smooth, longer-term trend
  100+ sessions: Very smooth, lag increases

TROUBLESHOOTING

Q: "No power hours found"
A: 1. Check data has correct timestamp index
   2. Verify power hour timing matches your market
   3. Use UTC conversion if data in different timezone
   4. Ensure intraday data (1H or lower), not daily

Q: Too many false breakouts?
A: 1. Increase sessions_memory (20→40)
   2. Require volatility >MODERATE before trading
   3. Add trend strength filter >70%
   4. Validate breakout with volume

Q: Trendlines too noisy?
A: 1. Increase sessions_memory
   2. Use lower timeframe aggregation
   3. Filter by trend consistency
   4. Add R² confidence minimum (>0.7)

Q: Levels not working?
A: 1. Validate touches (min 3 recommended)
   2. Check tolerance for "touching" level
   3. Use longer memory window
   4. Ensure clean historical data

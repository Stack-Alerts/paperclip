## 🎯 WHAT IS A STATISTICAL TRAILING STOP?

A **statistical trailing stop** is a dynamic stop-loss system that adjusts based on volatility distribution rather than fixed percentages or ATR multiples.

### Core Concept

Traditional trailing stops use fixed values. This indicator uses:
- **Log-normal distribution** of volatility
- **Four levels** from most responsive to most stable
- **Dynamic adjustment** based on volatility grouping
- **Statistical grounding** in market behavior

### How It Works

```
1. Group Candles (10 bars per group)
2. Calculate Volatility for Each Group
3. Fit Log-Normal Distribution (last 100 groups)
4. Generate 4 Levels:
   - Level 0: 0.8x ATR (most responsive)
   - Level 1: 1.2x ATR (standard)
   - Level 2: 1.6x ATR (stable) ← Most popular
   - Level 3: 2.0x ATR (very stable)
5. Signal on Bounce or Breakdown
```

### The Four Levels

| Level | Sensitivity | Use Case | ATR Multiple |
|-------|-------------|----------|--------------|
| **0** | Very High | Quick exits | 0.8x |
| **1** | High | Standard | 1.2x |
| **2** | Medium | Balanced | 1.6x |
| **3** | Low | Trend riding | 2.0x |

### 1. Volatility Analysis
```python
analyzer = VolatilityAnalyzer(data_length=10)

groups, df_extended = analyzer.group_volatility(df)
```

**What it does:**
- Groups consecutive bars (default: 10 bars)
- Calculates Garman-Klass volatility per group
- Fits log-normal distribution
- Provides statistical parameters

### 2. Trailing Stop Calculation
```python
detector = StatisticalTrailingStop()

# Calculate all levels
df_stops = detector.calculate_trailing_stops(df)

# Each bar now has 4 levels for long + 4 for short
print(df_stops[['level_0_long', 'level_1_long', 'level_2_long', 'level_3_long']])
```

**What it does:**
- Calculates ATR-based levels
- Adjusts for volatility distribution
- Provides both long (below) and short (above) stops
- Updates dynamically as market evolves

### 3. Signal Detection
```python
# Get all signals
signals = detector.detect_all_signals(df_stops, level=2)

# Long signals: Price bounces up through stop
for sig in signals['long']:
    print(f"Long @ {sig.timestamp}: {sig.price:.2f}")

# Short signals: Price bounces down through stop
for sig in signals['short']:
    print(f"Short @ {sig.timestamp}: {sig.price:.2f}")
```

**What it does:**
- Detects when price crosses trailing stop
- Distinguishes between long/short bounces
- Tracks volatility at signal point
- Provides entry signal data

### 4. Backtesting Engine

**What it does:**
- Entry on signal generation

- Calculates P&L and metrics

### Strategy 2: Responsive Scalping (Level 0)
```python
# Quick exits, frequent trades
signals = detector.detect_all_signals(df_stops, level=0)

# Tight stops, quick profits
for sig in signals['long']:
    risk = sig.price - sig.level_0
    target = sig.price + (risk * 1.5)
    print(f"Scalp: BUY {sig.price}, SL {sig.level_0}, TP {target}")
```

### Strategy 3: Trend Riding (Level 3)
```python
# Hold longer, less whipsaws
signals = detector.detect_all_signals(df_stops, level=3)

for sig in signals['long']:
    print(f"Hold trend: BUY {sig.price}")
    print(f"Wide stop: {sig.level_3}")
    print(f"Let profits run with level 3 protection")
```

### Strategy 4: Multi-Level Confluence
```python
# Only trade when multiple levels align
detector = StatisticalTrailingStop()
df_stops = detector.calculate_trailing_stops(df)

signals_l0 = detector.detect_long_signals(df_stops, level=0)
signals_l1 = detector.detect_long_signals(df_stops, level=1)
signals_l2 = detector.detect_long_signals(df_stops, level=2)

# Find signals where multiple levels trigger close together
for sig0 in signals_l0:
    for sig1 in signals_l1:
        time_diff = abs((sig0.timestamp - sig1.timestamp).total_seconds())
        if time_diff < 3600:  # Within 1 hour
            print(f"Multi-level confirmation at {sig0.timestamp}")
```

## ⚙️ PARAMETER GUIDE

### data_length (Volatility Grouping)
```
5-8 bars: Very responsive (more noise)
10 bars: Standard (balanced)
15-20 bars: Smoother (more lag)
```

**Default: 10 bars**

### distribution_length (Lookback)
```
50 groups: Short-term (recent conditions)
100 groups: Standard (most common)
200+ groups: Long-term (historical bias)
```

**Default: 100 groups**

### atr_period (Volatility Baseline)
```
7-10: Very responsive
14: Standard (two weeks)
21-28: Smoother (monthly)
```

**Default: 14**

### base_level Selection
```
Level 0: Scalping, tight stops, many signals
Level 1: Aggressive day trading
Level 2: Balanced approach (MOST POPULAR)
Level 3: Trend following, few signals
```

## 📊 DASHBOARD METRICS EXPLAINED

### Win Rate
- Percentage of trades that were profitable
- Example: 76.47% = 13 wins out of 17 trades

### Expectancy
- Average P&L per trade
- **Positive = Profitable system**
- Example: $4,839 per trade (excellent)

### Profit Factor
- Gross wins ÷ Gross losses
- >1.5 is good, >2.0 is excellent
- Example: 2.0 = Wins are 2x losses

### Sharpe Ratio
- Risk-adjusted returns
- Higher is better (>1.0 is good)
- Accounts for trade consistency

### Risk/Reward Ratio
- Average win ÷ Average loss
- Example: 3.34 = Win 3.34 for every 1 lost

### Max Drawdown
- Largest peak-to-trough decline
- Measures volatility in equity curve
- Lower is better

---

## 🎓 KEY CONCEPTS

### Log-Normal Distribution
- Volatility follows log-normal (not normal)
- Creates asymmetric distribution
- Better models real market behavior

### Volatility Grouping
- Aggregates bars into groups
- Smooths noise
- Creates statistical snapshots

### Dynamic Adjustment
- Stops adjust with market volatility
- Wide stops in volatile markets
- Tight stops in quiet markets

### Four Levels
- Level 0-3 provide spectrum of choices
- No single "best" level
- Choose based on trading style

## 🎯 WHAT IS MACD PRICE FORECASTING?

A forward-looking analysis tool that predicts price ranges after MACD signals using **historical trajectory analysis**.

### Core Concept

```
TRADITIONAL:
MACD signal → "Uptrend likely" → Guess what happens next

FORECASTING:
MACD signal → Analyze ALL past signals of this type →
Collect where price moved after → Calculate percentiles →
Project forward-looking range
```

### How It Works

1. **Detect MACD Signals**
   - MACD crosses above/below signal line
   - Or MACD crosses above/below zero line (configurable)

2. **Collect Historical Trajectories**
   - For every past MACD signal of same type
   - Record where price went for next N bars
   - Store up to max_memory signals

3. **Calculate Percentile Ranges**
   - Top percentile (95%) = aggressive upside target
   - Average percentile (50%) = expected midpoint
   - Bottom percentile (5%) = downside support

4. **Generate Forecast**
   - Lower bound (5th percentile) = support area
   - Middle line (50th percentile) = fair value
   - Upper bound (95th percentile) = resistance area

5. **Identify Reversals**
   - Signal type changes significantly = reversal likely
   - Used for swing trading entries/exits

### 1. MACD Calculation
```python
from luxalgo_macd_forecasting import MACDCalculator

calc = MACDCalculator(fast_length=12, slow_length=26, signal_length=9)
df_macd = calc.calculate_macd(df)

print(df_macd[['macd', 'signal', 'histogram']])
```

**What it does:**
- Fast EMA (12) - Slow EMA (26) = MACD line
- EMA of MACD = Signal line
- MACD - Signal = Histogram
- Standard technical analysis

### 2. Signal Detection
```python
from luxalgo_macd_forecasting import TrendDetermination

signals = calc.detect_signals(df_macd, TrendDetermination.MACD)

for sig in signals:
    print(f"{sig.timestamp}: {sig.signal_type.value}")
```

**Two methods:**
- **MACD**: MACD crosses zero line
- **MACD-Signal**: MACD crosses signal line

**What it does:**
- Identifies trend change points
- Bullish = MACD above signal
- Bearish = MACD below signal

### 3. Trajectory Collection
```python
trajectories = forecaster.collect_trajectories(df_macd, signals)

# trajectories['bullish'] = historical bullish outcomes
# trajectories['bearish'] = historical bearish outcomes
```

**What it does:**
- Records price movement after each signal
- Groups by signal type (bullish/bearish)
- Limited to max_memory to avoid old-regime bias

### 4. Forecast Generation
```python
forecast = forecaster.generate_forecast(df, signal, trajectories)

print(f"Upper: ${forecast.upper_bound:.2f}")
print(f"Lower: ${forecast.lower_bound:.2f}")
print(f"Range: ${forecast.confidence_range:.2f}")
```

**What it does:**
- Calculates percentiles from historical trajectories
- Projects range 20 bars into future (configurable)
- Provides entry/exit levels
- Measures confidence range

## 📈 TRADING STRATEGIES

### Strategy 1: Range Trading
```python
# Trade within forecast ranges
for forecast in forecasts:
    if forecast.signal_type == SignalType.BULLISH:
        print(f"BUY Support: ${forecast.lower_bound:.2f}")
        print(f"SELL Resistance: ${forecast.upper_bound:.2f}")
    else:
        print(f"SHORT Resistance: ${forecast.upper_bound:.2f}")
        print(f"COVER Support: ${forecast.lower_bound:.2f}")
```

### Strategy 2: Reversal Trading
```python
# Trade reversals when signal type changes
reversals = forecaster.identify_reversals(forecasts, reversal_threshold=0.02)

for rev in reversals:
    print(f"Reversal at {rev['timestamp']}")
    print(f"  From {rev['previous_signal']} to {rev['current_signal']}")
    print(f"  Entry: ${rev['curr_forecast'].signal_price:.2f}")
```

### Strategy 3: Breakout Confluence
```python
# Only trade when forecast aligns with other signals
for forecast in forecasts:
    # Combine with price action, volume, etc
    if forecast.confidence_range < 1.0:  # Narrow range = high confidence
        print(f"High confidence forecast: {forecast.signal_type.value}")
        print(f"  Range: ${forecast.confidence_range:.2f}")
```

### Strategy 4: Multi-Percentile Trading
```python
# Use different percentiles for different position sizing
for forecast in forecasts:
    # Aggressive entry at average
    aggressive_entry = forecast.average_price
    
    # Conservative entry at closer bound
    conservative_entry = forecast.middle_price
    
    # Tight stop at percentile
    tight_stop = forecast.bottom_percentile_price if forecast.signal_type == SignalType.BULLISH else forecast.top_percentile_price
    
    # Wide stop at bound
    wide_stop = forecast.lower_bound if forecast.signal_type == SignalType.BULLISH else forecast.upper_bound
```

## ⚙️ PARAMETER GUIDE

### MACD Parameters
```
Fast Length: 12 (standard, use 8-15)
Slow Length: 26 (standard, use 20-30)
Signal Length: 9 (standard, use 5-14)
```

### Forecast Parameters
```
max_memory: 100
  - Number of past signals to remember
  - Higher = smoother but potentially stale
  - Lower = more responsive but noisier
  
forecasting_length: 20
  - Bars ahead to forecast
  - Match your trading horizon
  
Percentiles: 95/50/5 (standard)
  - Top: 95th percentile = aggressive upside
  - Middle: 50th percentile = expected value
  - Bottom: 5th percentile = downside support
```

### Trend Determination
```
MACD: MACD > 0 = uptrend, < 0 = downtrend
  - Simpler, more lagging
  
MACD-Signal: MACD > Signal = uptrend, < = downtrend
  - More responsive, more signals
```

## 📊 FORECAST COMPONENTS EXPLAINED

### Signal Price
The close price at which the MACD signal occurred.

### Upper Bound (95th Percentile)
Maximum price historically reached after similar signals.
Used as aggressive profit target.

### Average Price (50th Percentile)
Typical price movement after signal.
Used as fair value or first target.

### Lower Bound (5th Percentile)
Minimum price historically reached.
Used as support or stop-loss area.

### Confidence Range
Upper bound - Lower bound = prediction width.
Narrow = high confidence
Wide = high uncertainty

### Top/Bottom Percentile Price
1-bar ahead percentile prices.
Shows immediate expected direction.

---

## 🎓 KEY CONCEPTS

### Percentile Analysis
- Based on historical outcomes after similar signals
- Not assuming normal distribution
- Captures actual market behavior
- Different for bullish vs bearish

### Trajectory Collection
- Remembers up to max_memory signals
- Prevents curve-fitting to old regimes
- Adapts to recent market conditions
- Trades off responsiveness vs stability

### Multiple Forecast Horizons
- Can forecast 1 bar, 5 bars, 20 bars ahead
- Short-term vs long-term forecasts
- Different confidence at different horizons

### Reversal Detection
- Signal type change = potential reversal
- Change in price range = weakening trend
- Useful for swing trade exits

## 📚 OPTIMIZATION TIPS

**For Day Trading:**
- Use shorter MACD lengths (10/20/5)
- forecasting_length = 10-15 bars
- max_memory = 50 (recent only)
- Tighter percentiles (90/50/10)

**For Swing Trading:**
- Standard MACD (12/26/9)
- forecasting_length = 20-30 bars
- max_memory = 100 (standard)
- Standard percentiles (95/50/5)

**For Position Trading:**
- Longer MACD lengths (15/30/9)
- forecasting_length = 50+ bars
- max_memory = 150+ (historical)
- Wider percentiles (98/50/2)

**For Reversal Trading:**
- Focus on signal type changes
- Lower reversal_threshold
- Use ConfidenceScorer for entries
- Combine with support/resistance

## How the Daemon Works

```
Every 5 minutes:
1. Query Paperclip API for done fix/bug issues (last 10 minutes)
2. Check each issue's comment history for existing gate results
3. For ungated issues:
   - Fetch touched files from issue description
   - Run full Impact Gate (FR + regression tests)
   - Post detailed result comment
   - Transition issue based on outcome (PASS→done, FAIL→in_progress)
4. Report metrics: issues_found, gated, skipped, errors
5. Sleep 300 seconds, repeat
```

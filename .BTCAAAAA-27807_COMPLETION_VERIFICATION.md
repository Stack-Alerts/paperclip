## How It Works

```
Every 5 minutes (via GitHub Actions):
  1. Trigger workflow: impact-gate-polling-daemon.yml
  2. Checkout repository and install dependencies
  3. Run: python scripts/impact_gate_polling_daemon.py --initial-scan
  4. Daemon:
     a. Queries Paperclip API for done fix/bug issues (last 10 min)
     b. Checks each issue's comments for gate results
     c. For ungated issues:
        - Reads touched files from description
        - Calls impact_gate.worker.process_issue() with force=True
        - Posts result comment (PASS/FAIL/SKIPPED/ERROR)
        - Creates blocking issues on FAIL
     d. Returns JSON summary: {timestamp, issues_found, gated, skipped, errors}
  5. GitHub Actions:
     - Renders step summary with metrics
     - Uploads JSON artifact
     - Creates CTO alert if ungated > 0
  6. Daily snapshot captures coverage metrics
  7. Muted state prevents re-opening done issues
```
